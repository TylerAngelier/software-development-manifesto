# Specs Directory

Approved and draft specs live here.

Guidelines:

- One spec file per feature or change set.
- Use `references/ai/templates/spec.md` as the template.
- Keep durable intent here: problem, scope, constraints, decisions, validation.
- Do not keep the authoritative task list here once Beads issues are created.
- Link Beads tasks back to the spec path with `--spec-id`.

Recommended naming:

- `auth-jwt.md`
- `billing-webhooks.md`
- `search-index-backfill.md`
